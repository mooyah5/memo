### 기능 목록
1. 임의의 todo app UI
2. 로그인/로그아웃 (토큰 기반)
3. todo 목록 불러오기 (GET)
4. todo 추가 (POST)
5. todo 삭제 (DELETE) 시 에러 핸들링
6. 로딩/에러 처리
7. 기타 비동기 관련 로직 만들어 해보기

### 요구 기술
- 전체 구조
    - 메인 서비스는 Express로 인증, API
    - 데이터 보관: 'node-persist' (직접 DB 구성도 됨)
    - 클라이언트는 vite + vinilla
- fetch, async/await 활용하여 구현
- 로딩 지연 구현
    - /todos 목록 응답 API는 고의로 2초 지연 응답
    - 2초 동안 'loading' 화면 표시
- Delete 서버 오류
    - /todos를 delete 요청 시에는 서버에서 삭제처리 않고 status 500 에러 반환
    - 클라이언트는 500 오류를 받아서 적절 에러 처리

### AI 활용
