# 블로킹과 논블로킹

### 제어권. 그리고 결과값을 기다린다는 것
블록과 논블록, 동기와 비동기를 이해하려면
다음 두 개념을 짚고 넘어가야 한다.

`제어권`은 자기 코드를 실행할 권리같은 것. (실행 흐름 제어 권리)
OS 커널에서 온 개념으로, I/O 동작 설명 시 주로 씀.
제어권을 가진 함수는 자기 코드를 끝까지 실행한 후,
자신을 호출한 함수에게 돌려줌.

`결과값을 기다린다는 것`은,
A가 B를 호출했을 때, A가 B의 결과값을 기다리는지 여부.


-----


### 블로킹과 논블로킹 (제어권을 넘겼는가)
A가 B를 호출했을 때,
**`제어권을 주느냐`** 안 주느냐.
 = 현재 **`작업이 대기`** 되는가?

##### 블로킹 (의존적)
A가 B를 호출하면 제어권을 넘겨줌.
B가 실행되는 동안 A는 멈춤.

##### 논블로킹 (진취적)
A가 B를 호출해도 제어권은 안 넘김.
B 호출 후에도 A는 계속 자기 코드 실행.


호출된 함수(callee)가 호출한 함수(caller)에게 제어권을 바로 주냐 마냐 (제어권 넘기면 -> 해당 스레드 블로킹됨.)

-----

### 동기와 비동기 (신경쓰는가)
A가 B를 호출한 뒤, B의 **`완료 여부를 신경쓰냐`** 안 쓰냐.
= 작업을 **`순차적으로 수행`** 하는가?

##### 동기 (불안형)
A가 B를 호출한 뒤, B의 리턴값을 계속 확인하며 신경씀.

##### 비동기 (회피형)
A가 B를 호출한 뒤, B의 작업 완료 여부에 신경쓰지 않음.

-----
### 비교

##### Sync-Blocking
A는 B 결과가 필요하니까(동기) 제어권 넘기고 기다림(블로킹)

**언제 쓰나**
- B 결과가 A 작업에 영향 줄 때
- 파일 읽고 내용 바로 처리할 때
- 간단하고 빠른 작업일 때

**단점**
- 작업이 오래 걸리면 그간 암것도 못함.
- 전체 처리 시간 증가 -> 이럴 땐 비동기 논블로킹!

##### Sync-Nonblocking
제어권은 안 주지만(논블로킹), 결과는 계속 확인(동기)

- **Java**
스레드로 작업을 백그라운드에 돌리지만,
메인에서 계속 "끝났어?" 하고 물어봄.

- **JS**
    ```
    async function sync-nonblock () {
        const res = await slowTask();   // 여기서 대기
        console.log(res)
    }
    ```
    - `await`로 비동기 작업을 순서대로 처리
    - 내부적으로는 이벤트 루프 돌지만, 다음 코드 결과 받을 때까지 실행 안 됨.
- **실행활**
    - 게임 로딩바: 로딩 중에도 바는 계속 움직이지만(논블로킹), 완료까지 다음 스테이지 못 감(동기)
    - 파일 다운로드: 다른 작업 가능하지만(논블로킹), 계속 진행률은 확인(동기)


##### Async-Nonblocking
제어권도 안 주고(논블로킹), 완료 여부도 신경 안 씀(비동기)
대신 `콜백함수`를 같이 보냄. B는 끝나면 콜백을 실행. (B의 완료에 대한 후처리 위해 사용됨)

**언제 쓰나**
- 다른 작업 결과가 내 작업에 영향 안 줄 때
- I/O처럼 느린 작업 시 안 기다리고 다른 일 하고 싶을 때

**예**
- 웹 브라우저 파일 다운로드
- `setTimeout`, API 호출

##### Async-Blocking
관심도 없는데(비동기), 제어권은 넘김(블로킹)...
관련 없는 작업 끝날 때까지 기다려야 함.

- 실무에서 거의 안 쓰임 (Sync-blocking과 성능 비슷)
- 의도치 않게 발생하는 경우가 대부분

-----

### 비동기와 논블로킹의 차이
**비동기:** '신경 안 씀'
**논블로킹:** '안 멈춤'

비동기는 결과를 기다리냐 마냐고,
제어권 넘기냐 마냐.

비동기는 계속 하려던 거 할지 말지가 아님.
논블로킹은 신경쓸지말지가 아님. 멈춤여부

**setTimeout**

```
console.log(1)
setTimeout(() => console.log(2), 0)
console.log(3)
// 1 3 2
```
- 비순차 실행 → **비동기**
- 메인 안 멈추고 백그라운드 처리 → **논블로킹**

-----
### 동기 처리와 멀티태스킹
**동시 처리:**: 둘 이상 작업 동시 실행
**구현 방식:**
- 멀티 스레드/프로세싱
- js는 브라우저에 내장된 멀티스레드 Web API에 작업을 위임함.
    - 타이머, 네트워크 통신, 이벤트 처리 등
    - 메인 콜 스택과 병렬처리됨.
