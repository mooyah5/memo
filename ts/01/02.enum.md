# enum

정해진 값들의 집합을 이름으로 관리하는 타입.
즉, 코드에서 쓸 수 있는 값들을 제한하고, 의미 잇는 이름을 붙여서 안전하게 쓰는 도구

### 사용 이유

-   값을 제한하고 실수 감소
    -   예: 신발 브랜드가 Nike, Adidas 중 하나여야 한다면, 문자열로 직접 쓰면 오타 위험 잇음
    -   enum 쓰면 IDE가 자동완성 해주고, 잘못된 값은 컴파일 에러가 남.
-   코드 가독성 향상
    -   `if (status === 1)` 보다 `if (status === Status.Success)`가 읽기 쉽져
-   상수 관리
    -   여러 곳에서 같은 값을 쓰면 나중에 바꾸기 힘듦
    -   enum으로 관리하면 한 곳에서 바꾸면 전체 반영됨.

### 구분

-   숫자 enum (default)
    -   값이 자동으로 0부터 증가함.
    -   그래서 Shoes.Nike는 0, Shoes.Adidas는 1이 됨
-   문자 enum
    -   각 멤버에 문자열 값을 직접 할당해야 함.
    -   자동 증가가 없고 숫자처럼 인덱스 기반이 아님
-   ts의 enum은 런타임에도 객체로 존재함. 숫자 이넘은 역방향 매핑(reverse mapping)이 있어 숫자 -> 이름도 가능.

```
    // case 1
    enum Shoes {
        Nike, Adidas
    }

    let myShoes = Shoes.Nike;
    console.log(myShoes);   // 0 (값을 안 주면 첫 멤버가 0, 이후 1씩 자동 증가한다. 그래서 index 처럼 보이는 할당된 숫자가 출력된다.)


    // case 2
    enum Shoes {
        Nike = 10,
        Adidas
    }

    let myShoes = Shoes.Adidas;
    console.log(myShoes);       // 11


    // case 3

    enum Shoes {
        Nike = 'nike',
        Adidas = 'adidas'
    }

    let myShoes = Shoes.Nike;
    console.log(myShoes)        // nike (문자열 enum은 반드시 직접 값을 할당해야 하고, 반환도 문자열임. 숫자이넘과 다르게 역방향 매핑이 없다. 문자열 -> 이름 불가)
```

java처럼 enum을 출력하면 0, 1, 2, 이런 식으로 현재 키의 index가 출력된다.
문자를 할당하여 추가로 추기화를 시켜줘야 한다.

### TS enum의 런타임 형태 (숫자 enum의 역방향 매핑)

-   숫자 enum은 런타임에 대충 다음처럼 컴파일 된다. (개념 설명용)

    ```
        var Shoes = {
            0: "Nike",
            1: "Adidas",
            Nike: 0,
            Adidas: 1
        }

        Shoes.Nike      // 0
        Shoes[0]        // "Nike" (역방향 매핑)
    ```

-   문자열 enum은 역방향 키가 없음
    ```
        var Shoes = { Nike: "nike", Adidas: "adidas" };
        // Shoes["nike"] 같은 역방향 접근은 불가
    ```

### 언제 어떤 enum을 쓰면 좋나?

-   숫자 enum
    -   빠르고 가볍게 `코드값`을 쓰고 싶을때
    -   역방향 매핑으로 숫자 -> 이름도 필요할 때
-   문자열 enum
    -   로그, 네트워크, 설정 파일 등에서 `값 자체가 의미를 가져야` 할 때
    -   사람이 ㅇ릭기 쉬운 값 필요 (예: 'content-type', 'adidas' 등)

### 실무 대체안 : 유니온 타입 + as const 객체

TS enum은 런타임 객체가 생성되므로 번들링/트리셰이킹 관점에서 부담이 될 수 있음.
nuxt3 + Vite/Webpack 환경이라면 다음 패턴을 많이 씀

```
    // 값 집합을 객체로 정의
    export const SHOES = {
        NIKE: 'nike',
        ADIDAS: 'adidas'
    } as const;

    // 타입은 키가 아니라 값 유니온으로 만들기
    export type Shoe = typeof SHOES[keyof typeof SHOES];    // 'nike' | 'adidas'

    const myShoe: Shoe = SHOES.NIKE // 'nike'
```
